# Contract Note Template

Use a compact note. Keep it close to implementation and testing needs.

```markdown
## Frontend Data Contract Note

- Consumer:
- Existing mock/static source:
- Provider type:
- Provider evidence:
- Provider call/path:
- Input mapping:
- Response/data mapping:
- Adapter needed:
- State behavior:
- Auth/client setup:
- Contract status: pass / partial / blocked
- Missing information:
- Verification evidence:
```

For data-service work, `Provider evidence` can point to API文档/API清单. For non-data-service work, point to SDK docs, schema files, local data modules, fixtures, or runtime traces.
